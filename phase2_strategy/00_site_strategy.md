# Site Strategy

> Author: Codex
> Date: 2026-04-02

## Product Position

KBFA website V1 should be positioned as:

an official, bilingual association website that builds trust, showcases achievements, supports partnership inquiries, and opens a practical path for sponsorship.

## Primary User Groups

1. General visitors seeking to understand the association
2. Potential partners such as institutions, tourism organizations, cultural groups, and regional bodies
3. Potential sponsors and supporters
4. Media or event stakeholders checking organizational credibility

## Priority Outcomes

1. Visitors understand who KBFA is within seconds
2. Visitors can quickly see what KBFA has actually done
3. Partners can find contact and collaboration routes easily
4. Supporters can move from interest to sponsorship without confusion

## Strategic Differentiation

KBFA should not present itself as:

- a generic travel promotion site
- a static brochure page
- a donation page without institutional context

KBFA should present itself as:

- a Korea-Bhutan friendship bridge
- a credible organizer and collaborator
- a mission-led cultural and partnership platform

## Strategic Page Groups

### 1. Trust Layer

- Home
- About KBFA
- History
- Partners

### 2. Proof Layer

- Major Activities
- Timeline or portfolio highlights
- News and announcements

### 3. Action Layer

- Contact
- Project sponsorship
- One-time sponsorship

## Sponsorship Strategy

### Project Sponsorship

Use project pages when:

- the campaign has a specific purpose
- fund usage can be described clearly
- progress visibility helps motivate support

### One-Time Sponsorship

Use one-time sponsorship when:

- the supporter wants to help the association generally
- there is no need to choose a specific project

## Content Operations Strategy

For V1, content should be edited via structured Markdown and asset folders.

Why:

- faster launch
- easier owner control
- less dependency on a complex CMS
- future migration path remains open

## Email Strategy

- Inquiry form should send to the association inbox
- Sponsorship flow should send confirmation and transfer guidance recap to the donor
- English and Korean templates should be managed separately

## Success Indicators

- visitors can find core trust information quickly
- notices and activities can be updated without code rewrites
- sponsorship pages are clear enough to use without manual explanation
- bilingual structure remains maintainable over time
